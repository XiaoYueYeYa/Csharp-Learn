## 什么是编译器
编译器是一种翻译程序
它用于将语言程序翻译为目标语言程序

源语言程序:某种程序设计语言写成的 比如C# C C++ Java等语言写的程序
目标语言程序:二进制数表示的伪机器代码写的程序

## 什么是预处理器指令
预处理器指令 指导编译器 在实际**编译开始之前对信息进行预处理**
预处理器指令 都是**以#开始**
预处理器指令**不是语句** 所以它们**不以分号;结束**
目前我们常用到的 "折叠代码块" 就是预处理器指令

### 常见的预处理器指令
1.
\#define
定义一个符号 类似一个没有值的变量
\#undef
取消define定义的符号 让其失效
两者都是写在脚本文件最前面
一般配合 if指令使用 或配合特性

2.
\#if
\#elif
\#else
\#endif  用于结尾结束
和if语句规则一样 一般配合#define定义的符号使用
用于告诉编译器进行编译代码的流程控制

如果发现有#define的符号 那么其中包含的代码 就会被编译器翻译 (执行)
可以通过 逻辑或|| 和 逻辑与&& 进行多种符号组合判断
```C#
//定义一个符号
#define Unity4
#define Unity5
#define Unity6
#define Android
#define IOS
#define PC
//取消定义一个符号
#undef Unity4

```

```C#
#if Unity4
        Console.WriteLine("Unity4版本");
        #error 这个版本不能执行
#elif Unity5
        Console.WriteLine("Unity5版本");
		#warning 这个版本不合法
        
#elif Android || IOS
        Console.WriteLine("Android和IOS版本");
#else
        Console.WriteLine("其他版本");
#endif
```

3.
\#warning
\#error
告诉编译器
是报警告还是报错误
一般还是配合if使用

## 总结
总结:
预处理器指令
可以让代码还没有编译之前就可以进行一下预处理判断
在Unity中会用来进行一些平台或者版本的判断
决定不同的版本或者不同的平台使用不同的代码逻辑

## 练习
请使用预处理器指令实现
写一个函数计算两个数
当是Unity5版本时 算加法
当是Unity2017版本时 算乘法
当是Unity2020版本时 算减法
都不是返回0

```C#
#define Unity5
#define Unity2017
#define Unity2020
```

```C#
CC(2, 3);
```

```C#
static void CC(int a,int b)
{
	#if Unity5
	        Console.WriteLine("Unity5:"+(a + b));
	#endif
	#if Unity2017
	        Console.WriteLine("Unity2017:" + (a * b));
	#endif
	#if Unity2020
	        Console.WriteLine("Unity2020:" + (a - b));
	#endif
	        Console.WriteLine(0);
}
```
